# 电商合规哨兵 - 部署指南

## 1. 环境要求

### 1.1 硬件要求

**开发环境最低配置：**
- CPU: 4核心
- 内存: 8GB RAM
- 存储: 50GB SSD
- 网络: 稳定的互联网连接

**生产环境推荐配置：**
- CPU: 8核心及以上
- 内存: 16GB RAM及以上
- 存储: 200GB SSD（支持高IOPS）
- 网络: 100Mbps带宽，建议使用云服务器

### 1.2 软件要求

**核心依赖：**
- Python 3.9+（推荐3.11）
- PostgreSQL 14+
- Redis 7+
- Elasticsearch 8.x
- Neo4j 5.x

**操作系统：**
- Linux: Ubuntu 22.04 LTS / CentOS 8+ / Debian 11+
- Windows: Windows 10/11 或 Windows Server 2019+
- macOS: 12.0 Monterey及以上

### 1.3 网络要求

- 开放端口：
  - 8000: API服务
  - 5432: PostgreSQL
  - 6379: Redis
  - 9200: Elasticsearch
  - 7474/7687: Neo4j HTTP/Bolt协议
  - 9090: Prometheus（可选）
  - 3000: Grafana（可选）

## 4. 生产环境部署

### 4.1 Docker部署方案

**Dockerfile：**

```dockerfile
FROM python:3.11-slim

WORKDIR /app

# 安装系统依赖
RUN apt-get update && apt-get install -y \
    gcc \
    postgresql-client \
    && rm -rf /var/lib/apt/lists/*

# 安装Python依赖
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 复制应用代码
COPY . .

# 暴露端口
EXPOSE 8000

# 启动命令
CMD ["uvicorn", "src.api:app", "--host", "0.0.0.0", "--port", "8000"]
```

**Docker Compose配置：**


### 4.3 Nginx反向代理配置

```nginx
upstream mirolaw_backend {
    least_conn;
    server 127.0.0.1:8000 weight=5;
    server 127.0.0.1:8001 weight=5;
    server 127.0.0.1:8002 weight=5;
}

server {
    listen 80;
    server_name api.mirolaw.com;
    
    # 重定向到HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name api.mirolaw.com;

    # SSL证书配置
    ssl_certificate /etc/nginx/ssl/mirolaw.crt;
    ssl_certificate_key /etc/nginx/ssl/mirolaw.key;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    # 安全头
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;

    # 请求体大小限制
    client_max_body_size 10M;

    location / {
        proxy_pass http://mirolaw_backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # 超时配置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # API文档
    location /docs {
        proxy_pass http://mirolaw_backend/docs;
    }

    location /redoc {
        proxy_pass http://mirolaw_backend/redoc;
    }
}
```

## 5. 数据库配置

### 5.1 PostgreSQL初始化脚本

```sql
-- scripts/init_postgres.sql

-- 创建扩展
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
CREATE EXTENSION IF NOT EXISTS "pg_trgm";

-- 创建表
CREATE TABLE IF NOT EXISTS legal_cases (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    case_id VARCHAR(100) UNIQUE NOT NULL,
    title TEXT NOT NULL,
    content TEXT,
    risk_type VARCHAR(50),
    penalty_amount DECIMAL(15, 2),
    publish_date DATE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS articles (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    article_id VARCHAR(100) UNIQUE NOT NULL,
    title TEXT NOT NULL,
    content TEXT,
    category VARCHAR(100),
    effective_date DATE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS risk_predictions (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    case_id UUID REFERENCES legal_cases(id),
    risk_type VARCHAR(50),
    probability DECIMAL(5, 4),
    prediction_horizon INTEGER,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 创建索引
CREATE INDEX idx_cases_risk_type ON legal_cases(risk_type);
CREATE INDEX idx_cases_publish_date ON legal_cases(publish_date);
CREATE INDEX idx_articles_category ON articles(category);
CREATE INDEX idx_predictions_case_id ON risk_predictions(case_id);

-- 创建更新时间触发器
CREATE OR REPLACE FUNCTION update_updated_at_column()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = CURRENT_TIMESTAMP;
    RETURN NEW;
END;
$$ language 'plpgsql';

CREATE TRIGGER update_legal_cases_updated_at BEFORE UPDATE ON legal_cases
    FOR EACH ROW EXECUTE FUNCTION update_updated_at_column();
```

### 5.3 Neo4j知识图谱导入

```cypher
// 创建约束和索引
CREATE CONSTRAINT case_unique IF NOT EXISTS FOR (c:LegalCase) REQUIRE c.case_id IS UNIQUE;
CREATE CONSTRAINT article_unique IF NOT EXISTS FOR (a:Article) REQUIRE a.article_id IS UNIQUE;
CREATE CONSTRAINT risk_type_unique IF NOT EXISTS FOR (r:RiskType) REQUIRE r.name IS UNIQUE;

// 创建风险类型节点
CREATE (:RiskType {name: '虚假宣传', severity: 3});
CREATE (:RiskType {name: '价格欺诈', severity: 4});
CREATE (:RiskType {name: '知识产权侵权', severity: 5});
CREATE (:RiskType {name: '不正当竞争', severity: 3});
CREATE (:RiskType {name: '消费者权益侵害', severity: 4});

// 导入法律案例（从CSV）
LOAD CSV WITH HEADERS FROM 'file:///legal_cases.csv' AS row
CREATE (c:LegalCase {
  case_id: row.case_id,
  title: row.title,
  risk_type: row.risk_type,
  penalty_amount: toFloat(row.penalty_amount),
  publish_date: date(row.publish_date)
});

// 创建关系：案例-风险类型
MATCH (c:LegalCase), (r:RiskType)
WHERE c.risk_type = r.name
CREATE (c)-[:HAS_RISK_TYPE]->(r);

// 创建关系：案例-法律条文
LOAD CSV WITH HEADERS FROM 'file:///case_article_relations.csv' AS row
MATCH (c:LegalCase {case_id: row.case_id})
MATCH (a:Article {article_id: row.article_id})
CREATE (c)-[:REFERENCES_ARTICLE {relevance: toFloat(row.relevance)}]->(a);
```

## 7. 备份与恢复

### 7.1 数据备份策略

**PostgreSQL备份：**

```bash
#!/bin/bash
# scripts/backup_postgres.sh

DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="/backup/postgres"
mkdir -p $BACKUP_DIR

# 全量备份
pg_dump -U postgres -d mirolaw -F c -f "$BACKUP_DIR/mirolaw_$DATE.backup"

# 保留最近7天的备份
find $BACKUP_DIR -name "*.backup" -mtime +7 -delete

# 上传到云存储（可选）
# aws s3 cp "$BACKUP_DIR/mirolaw_$DATE.backup" s3://your-bucket/backups/
```

**定时任务 (crontab)：**

```cron
# 每天凌晨2点备份

# 每小时增量备份

```

```bash
{
  "type": "fs",
  "settings": {
    "location": "/backup/elasticsearch"
  }
}
'

# 创建快照
curl -X PUT "localhost:9200/_snapshot/backup_repo/snapshot_$(date +%Y%m%d)"
```

**Neo4j备份：**

```bash
# 在线备份
neo4j-admin backup --from=bolt://localhost:7687 --backup-dir=/backup/neo4j

# 或使用dump命令（需停止服务）
neo4j-admin database dump neo4j --to=/backup/neo4j/neo4j_$(date +%Y%m%d).dump
```

### 7.2 数据恢复流程

**PostgreSQL恢复：**

```bash
# 恢复数据库
pg_restore -U postgres -d mirolaw_new -F c /backup/postgres/mirolaw_20240101.backup

# 或使用SQL文件
psql -U postgres -d mirolaw < /backup/postgres/mirolaw_20240101.sql
```

**Elasticsearch恢复：**

```bash
# 关闭索引
curl -X POST "localhost:9200/legal_cases/_close"

# 恢复快照
curl -X POST "localhost:9200/_snapshot/backup_repo/snapshot_20240101/_restore"

# 打开索引
curl -X POST "localhost:9200/legal_cases/_open"
```

**Neo4j恢复：**

```bash
# 停止服务
neo4j stop

# 恢复数据
neo4j-admin database load neo4j --from=/backup/neo4j/neo4j_20240101.dump

# 启动服务
neo4j start
```

## 8. 常见问题

### 8.1 FAQ

**Q1: 服务启动失败，提示数据库连接错误？**

A: 检查以下项：
1. 数据库服务是否启动
2. 连接参数是否正确（host、port、用户名、密码）
3. 防火墙是否开放端口
4. 数据库是否已创建

**Q2: Elasticsearch启动报错"memory locking requested for elasticsearch process but memory is not locked"？**

A: 修改系统配置：

```bash
sudo sysctl -w vm.max_map_count=262144
# 永久生效
echo "vm.max_map_count=262144" | sudo tee -a /etc/sysctl.conf
```

**Q3: Neo4j浏览器无法访问？**

A: 检查配置文件 `conf/neo4j.conf`：

```properties
# 启用HTTP访问
dbms.connector.http.enabled=true
dbms.connector.http.listen_address=:7474

# 允许远程访问
dbms.connector.http.advertised_address=:7474
```

**Q4: API响应速度慢？**

A: 优化建议：
1. 增加Redis缓存使用
2. 优化数据库查询，添加索引
3. Elasticsearch使用分页查询
4. Neo4j优化查询语句，使用PROFILE分析
